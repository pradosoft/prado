
# Working Knowledge - Knowledge Retention Sub-Process Instructions
- The point of "Working Knowledge" is to make Coding Agents more efficient at project tasks.
- The Working Knowledge directory is "agents/"
- The Working Knowledge directory for "framework/" is "agents/framework/"
- Working Knowledge is contained in "LIBRARY.md" files.
- Working Knowledge for "framework" is mapped to "agents/framework/" with the directory hierarchy in the file name and '-' between the directories, and file; in SCREAMING-KEBAB-CASE. eg "agents/framework/WEB-UI-WEBCONTROLS-LIBRARY.md" 
- Scan files (via "find agents/ -type f -name '*.md'") and parse relevant knowledge files relating to directories and tasks at hand
- When class files are updated its directory Working Knowledge file must be updated.
- When a directory Working Knowledge file is updated, the parent directory knowledge files must be recursively updated.
