# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Purpose

IBM DB2 (LUW) driver-specific implementations of the database metadata and query-builder abstractions.

Requires the `pdo_ibm` PHP extension. PDO driver name: `ibm`.

## Classes

- **`TIbmMetaData`** — Extends `TDbMetaData`. Queries `SYSCAT.COLUMNS`, `SYSCAT.TABLES`, `SYSCAT.KEYCOLUSE`, `SYSCAT.TABCONST`, and `SYSCAT.REFERENCES` to build table/column metadata. Default schema resolved via `VALUES CURRENT SCHEMA`. Normalises all identifiers to uppercase for catalog queries, returns lowercase IDs to PHP.

- **`TIbmTableInfo`** — Extends `TDbTableInfo`. Stores schema name separately. `getTableFullName()` returns `"SCHEMA"."TABLE"` (double-quote delimited).

- **`TIbmTableColumn`** — Extends `TDbTableColumn`. Maps DB2 types (`INTEGER`, `BIGINT`, `SMALLINT`, `DECIMAL`, `NUMERIC`, `DOUBLE`, `REAL`, `FLOAT`, `DECFLOAT`, `BOOLEAN`, `CHARACTER`, `VARCHAR`, `CLOB`, `BLOB`, `DATE`, `TIME`, `TIMESTAMP`, `XML`) to PHP types. `getAutoIncrement()` reflects the DB2 `IDENTITY` column attribute.

- **`TIbmCommandBuilder`** — Extends `TDbCommandBuilder`. DB2-specific row limiting: `FETCH FIRST n ROWS ONLY` for limit-only; `ROW_NUMBER() OVER()` subquery for limit + offset (compatible with DB2 LUW 9.x+). `getLastInsertID()` uses `SELECT IDENTITY_VAL_LOCAL() FROM SYSIBM.SYSDUMMY1`.

## Conventions

- DB2 uses double-quotes `"` for case-sensitive identifier quoting; unquoted identifiers are uppercase.
- The default schema is the connected user's schema (`CURRENT SCHEMA`). Override with `setDefaultSchema()`.
- `GENERATED ALWAYS AS IDENTITY` / `GENERATED BY DEFAULT AS IDENTITY` columns map to `getAutoIncrement() === true`.
- DB2 does not support `LIMIT`/`OFFSET` syntax; use `TIbmCommandBuilder` which handles the `ROW_NUMBER()` rewrite automatically.
- `XML` columns are returned as strings by PDO; type is mapped to `string` in PHP.
- Views are distinguished from tables via `SYSCAT.TABLES.TYPE = 'V'`.
